# 系统提示词使用指南

## 概述

系统提示词功能允许你为AI助手设定特定的角色和工作方式。默认使用"产品专家"角色，非常适合进行AI直播产品的分析与调研工作。

## 快速开始

### 查看当前提示词

```bash
你: prompt
```

输出示例：
```
📝 当前系统提示词:
类型: product_expert
名称: 产品专家（默认）

内容预览:
你是一位资深的AI直播产品专家，目前正在从零开始搭建一个AI数字人直播产品...
```

### 查看所有可用提示词

```bash
你: prompt list
```

### 切换提示词

```bash
你: prompt tech_expert
```

系统会提示切换成功，并建议清空对话历史以应用新提示词。

## 可用的系统提示词

### 1. product_expert（产品专家 - 默认）

**适用场景**：产品规划、市场调研、竞品分析

**特点**：
- 从商业、市场、用户、企业、产品五个维度进行结构化分析
- 提供可落地的产品建议
- 结合当前AI直播项目背景
- 可访问本地知识库

**分析框架**：
```
1. 商业维度 - 商业模式、定价策略、市场规模
2. 市场维度 - 目标市场、竞争格局、差异化
3. 用户维度 - 用户画像、需求痛点、使用场景
4. 企业维度 - 资源能力、组织架构、技术实现
5. 产品维度 - 功能设计、产品形态、迭代规划
```

**使用示例**：
```
你: 分析智播AI的商业模式和定价策略
你: 对比淘宝AI直播的三个典型产品
你: 我们的产品应该如何定位？
```

### 2. pure_chat（纯聊天模式）

**适用场景**：日常对话、通用问答

**特点**：
- 无特定角色设定
- 通用AI助手
- 适合各类问题

**使用示例**：
```
你: 今天天气怎么样？
你: 帮我写一段Python代码
```

### 3. tech_expert（技术专家）

**适用场景**：技术选型、架构设计、代码审查

**特点**：
- 技术深度分析
- 提供代码示例
- 考虑性能和可维护性
- 给出具体技术方案

**使用示例**：
```
你: 如何优化视频生成的性能？
你: 帮我设计一个高并发的直播系统架构
你: 这段代码有什么问题？
```

### 4. data_analyst（数据分析师）

**适用场景**：数据分析、报表生成、趋势预测

**特点**：
- 数据驱动决策
- 提供可视化建议
- 揭示数据洞察
- 给出业务建议

**使用示例**：
```
你: 分析这组数据的趋势
你: 如何提升用户留存率？
你: 请帮我设计一个数据看板
```

### 5. none（无系统提示词）

**适用场景**：完全自由对话

**特点**：
- 不使用任何系统提示词
- AI按原始能力回答
- 最大灵活性

**使用示例**：
```
你: prompt none
```

## 最佳实践

### 1. 根据任务选择合适的角色

```bash
# 做产品分析时
你: prompt product_expert
你: 分析智播AI的竞争优势

# 写技术方案时
你: prompt tech_expert
你: 设计视频生成的API接口

# 做数据分析时
你: prompt data_analyst
你: 分析用户行为数据
```

### 2. 切换提示词后清空历史

切换提示词后，建议使用 `clear` 命令清空对话历史，这样新对话会完全应用新的提示词：

```bash
你: prompt tech_expert
✅ 已切换到: 技术专家
💡 提示: 建议使用 'clear' 命令清空对话历史，以应用新提示词

你: clear
✅ 当前会话历史已清空（保留系统消息）
```

### 3. 查看完整提示词内容

```bash
# 查看当前提示词的完整内容
你: prompt
```

会显示前300字符的预览，可以了解AI的角色设定。

### 4. 自定义提示词（开发者）

如果需要添加新的系统提示词，可以编辑：
```
component/chat/config/system_prompts.py
```

添加新的提示词到 `SystemPrompts` 类中。

## 提示词工作原理

### 系统消息注入

系统提示词会作为第一条消息注入到对话中：

```python
messages = [
    {"role": "system", "content": "你是一位资深的AI直播产品专家..."},  # 系统提示词
    {"role": "user", "content": "分析智播AI"},                         # 用户输入
    {"role": "assistant", "content": "智播AI是..."},                  # AI回复
]
```

### 提示词优先级

1. **系统提示词** > 用户输入
2. 系统提示词会影响AI的所有回答
3. 用户可以通过输入覆盖部分行为，但核心角色不变

### 提示词生效时机

- **新对话**：立即生效
- **现有对话**：需要清空历史后生效（系统消息在第一条）

## 配置文件

### agent_config.py

```python
# 默认使用的提示词类型
DEFAULT_SYSTEM_PROMPT_TYPE = "product_expert"

# 是否默认启用系统提示词
ENABLE_SYSTEM_PROMPT = True
```

### system_prompts.py

包含所有可用的系统提示词定义：
- `DEFAULT_PROMPT` - 默认产品专家提示词
- `PURE_CHAT_PROMPT` - 纯聊天提示词
- `TECH_EXPERT_PROMPT` - 技术专家提示词
- `DATA_ANALYST_PROMPT` - 数据分析师提示词

## 常见问题

### Q1: 切换提示词后为什么回答还是按旧角色？

A: 因为对话历史中的旧系统消息仍在生效。使用 `clear` 命令清空历史：

```bash
你: clear
```

### Q2: 如何完全禁用系统提示词？

A: 使用 `prompt none` 命令：

```bash
你: prompt none
```

### Q3: 可以同时使用多个提示词吗？

A: 不可以，每次只能使用一个系统提示词。但你可以随时切换。

### Q4: 提示词会影响工具调用吗？

A: 不会。工具调用（知识库、网络访问）与提示词独立工作。

### Q5: 如何查看所有命令？

A: 使用 `help` 命令查看完整命令列表：

```bash
你: help
```
